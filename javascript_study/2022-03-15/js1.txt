* Java와 Javascript 차이

자바 : 객체지향 프로그래밍 언어.거의 모든 환경에서 활용되며, 다양한 운영체제에서 실행될 수 있는 독립적 언어.

자바스크립트 : 객체지향 스크립트 언어. 웹 브라우저와 운영체제에서 활용할 수 있는 언어. 사용하기 위해서는 반드시 CSS와 HTML이 필요하다.

자바는 바다, 자바스크립트는 시냇물? 졸졸졸ㅋㅋ

- 웹 애플리케이션의 장점 : 웹 브라우저만 있으면 데스크톱, 노트북, 스마트폰 등 모든 장치에서 사용이 가능하다.

- HTML5 도입에 따라 웹 상에서 플래시 도움 없이 복잡한 프로그램 제작 가능

- 자바스크립트 코드 위치
기본 페이지의 head 태그 사이에 script 태그 삽입

- HTML5에서는 script 태그에 type 속성 적지 않기로 함

- 외부 스크립트 파일 넣기
<script src="파일명.js"></script>
CSS 외부파일 링크와 마찬가지로 .js 파일로 생성이 가능

- 자바스크립트 작동의 원리
웹 브라우저에서 순차적으로 실행.
head에 script를 넣으면 body 태그 읽기 전 실행됨.
script를 head 안에 넣어야 코드를 살펴보기 편리함.
but, 속도를 따진다면 body 끝나는 시점에 넣는 게 가장 유리하다.

- 표현식이란?
값을 만들어내는 간단한 코드
ex) 263
10 + 20 + 30
'혜연'

- 문자이란?
하나 이상의 표현식이 모인 것
문장이 모여 프로그램 구성
문장의 끝에는 항상 세미콜론을 찍어 문장의 종결을 알려준다.
alert('hello javascript');

- 키워드란?
자바스크립트가 처음 만들어질 때부터 정해진 특별한 의미가 있는 단어

- 식별자
자바스크립트에서 이름을 붙일 때 사용
ex) 변수명, 함수명
변수란? 값을 담는 공간
키워드 사용 불가, 숫자 시작 불가, 특수분자는 _과 $만 허용. 공백문자 사용 불가.

함수란? 특정한 목적이 있는 기능을 만들기 위한 프로그래밍을 담아 놓은 공간

- 식별자 생성 규칙
알파벳 사용기 관레
input, output같은 의미 있는 단어 사용
생성자 함수 이름은 항상 대문자로 시작 (?)
변수, 인스턴스, 함수, 메서드 이름은 항상 소문자로 시작
여러 단어로 이뤄진 각식별자는 각 단어의 첫 글자를 대문자로.

(카멜 표기법)
will out -> willOut
will retruen -> willReturn
i am a boy -> iAmABoy

- 식별자 종류
식별자 뒤 괄호 X : 변수, 다른 식별자랑 함께 쓰이면 속성
식별자 뒤 괄호 O : 함수, 다른 식별자랑 함께 쓰이면 메서드

alert('Hello world') 함수
Array.length 속성
input 변수
promt ('Message', 'Defstr') 함수
Math.PI 속성
Math.abs(-273) 메서드
메서드 : 함수 안에 함수! 

- 자바스크립트 주석 표현
 //를 사용함 (// 뒤 문장은 실행되지 않음)

 /* */을 사용해 여러 줄 주석 표현 가능

주석은 코드의 실행에 아무 영향을 미치지 않는다.
