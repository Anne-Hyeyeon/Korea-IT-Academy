- 리액트는 원래 페이스북이라는 회사에서 자기네들끼리 쓰기 위해 만든 기술임.
- 페북, 인스타, 에어비앤비 등... 많은 회사들이 리액트를 사용하고 있음
- 리액트는 프레임워크임
- 여기에 나오는 자바스크립트는 지금까지 배운 것과 폼이 다르다. 

리액트 사용규칙
(http://standardjs.com/)

- 작은 따옴표 사용하기
- but, 변수명 입력할 때는 ` 사용도 가능하다. html 지정시 "도 가능하다. 하지만 일반 스트링의 경우 '을 사용하는 게 좋다.
- 함수 이름을 쓸 때 한 칸을 띄워서 만들어야 한다.
function () 요러케. function() 하지 말고!
- 비교문을 쓸 때 === 를 사용한다.(형태까지 같은 것) ==는 사용하지 말 것.
- infix operaters -> 한칸씩 다 스페이스를 넣어서 띄어 쓰자.
- 배열의 경우에도 한 칸씩 띄어서 쓰자. ex) [1, 2, 3, 4]
- 세미콜론을 뒤에 붙일 필요가 없음.
-  if else 쓸 때, } else { 요렇게 쓸 것.
- if문을 쓸 때 중괄호 브라켓을 사용할 필요가 없어졌다. 한줄로 처리하면 인식이 됨. 단, 브라켓 생략하려면 두 줄로 써서는 안된다.
- 변수 선언 후 출력할때까지, 의미 없이 빈 줄을 쓰지 말 것.
- 콜론 들어간 문장 줄바꿈 할 때, 두 번째 줄에 붙일것. 콜론도 함께 줄바꿈
- 브라켓 안에 스페이스를 둘 것 ex) function kim ( return true )
- 변수명, 함수명 쓸 때 카멜케이스 꼭 사용!
- 콤마는 다음 줄이 아닌 첫째 줄의 마지막에 쓴다.
- 반대로, 콤마는 다음 줄 첫머리에 붙여준다.
- func-call-spacing 찾아보기
- 객체 만들 때 : 키값에다가 콜론 : 붙여누고, 값 앞에는 스페이스를 하나 둬라.
- 생성자 이름은 대문자로 시작해야 한다.
- 생성자를 만들 땐, 괄호를 반드시 넣어줘야 한다.
- new Array 사용 x, 꺾쇠 사용하기
- const로 변수를 넣었을 때, 값을 재선언 하지 말것. (const는 상수이기 때문에. 변하지 않는다.)
- window는 기본 변수이기 때문에, 할당하지 말 것.
